## 1. プロジェクト概要

### 1.1 プロジェクト名
手作り弁当喰楽部 鉄人 公式Webサイト フルリニューアル

### 1.2 目的
- 売上アップへの貢献
- モダンで食欲をそそるデザインへの刷新
- 注文導線の改善（電話番号への明確な誘導）
- ブランドイメージの向上

### 1.3 現状の課題
- デザインが古く、食欲訴求が弱い
- 電話番号への導線がわかりにくい
- 配達エリアがPDFのみで視覚的にわかりにくい
- モバイル対応が不十分

## 4. 機能要件

### 4.1 ご注文機能（電話注文）

**注文方法: 電話のみ**

#### 4.1.1 電話注文への誘導
- 電話番号を大きく目立つように表示
- スマホでタップして発信できるボタン（`tel:` リンク）
- 各ページから電話番号へアクセスしやすい導線

#### 4.1.2 ご注文ページの内容

| セクション | 内容 |
|------------|------|
| 電話番号 | **03-3432-3720**（タップで発信） |
| 受付時間 | 平日 9:00〜14:00、16:30〜21:00 / 土曜 〜20:00 |
| 注文の流れ | 1. メニューを選ぶ → 2. 電話で注文 → 3. お届け |
| 注文時に伝えること | お名前、電話番号、届け先住所、希望時間、弁当名・個数 |
| 支払い方法 | 現金 / QRコード決済 |
| 配達について | 注文から約40分〜、1個から配達可能 |

#### 4.1.3 電話ボタンUI（イメージ）

```
┌─────────────────────────────────────────┐
│  📞 電話で注文する                        │
│  03-3432-3720                           │
│  タップで発信                            │
└─────────────────────────────────────────┘
```

※ヘッダー・フッターにも電話番号を常時表示

### 4.2 配達エリア表示機能

| 機能 | 詳細 |
|------|------|
| **エリア一覧表** | 区ごとに地名・最低注文金額を表形式で表示 |
| **配達時間目安** | 最低金額帯ごとの配達所要時間を表示 |
| **エリア外対応** | 前日予約で15,000円〜対応可能な旨を記載 |

※ 地図表示は行わない（シンプルなテーブル形式のみ）

### 4.3 お知らせ機能

| 機能 | 詳細 |
|------|------|
| **一覧表示** | 日付・タイトル・カテゴリでリスト表示 |
| **詳細表示** | 本文・画像を表示 |
| **カテゴリ** | 新メニュー/休業日/キャンペーン/その他 |
| **更新頻度** | 月1〜2回程度 |
| **更新担当** | 店舗スタッフ（JSONファイル編集） |
| **更新方法** | JSONファイルで管理（運用マニュアル付き） |

### 4.4 求人ページ

| 項目 | 詳細 |
|------|------|
| **募集職種** | 配達スタッフ、調理スタッフ |
| **掲載内容** | 仕事内容、待遇、勤務時間、応募資格、応募フロー |
| **応募方法** | 電話のみ（03-3432-3720） |
| **更新頻度** | 募集状況に応じて随時 |

### 4.5 その他機能

| 機能 | 詳細 |
|------|------|
| **電話発信** | スマホでタップして電話発信 |

## 5. 非機能要件

### 5.1 パフォーマンス

| 指標 | 目標値 |
|------|--------|
| First Contentful Paint | 1.5秒以下 |
| Largest Contentful Paint | 2.5秒以下 |
| Cumulative Layout Shift | 0.1以下 |
| PageSpeed Insights | モバイル90点以上 |

### 5.2 対応環境

| 項目 | 対応範囲 |
|------|----------|
| **デバイス** | PC、タブレット、スマートフォン |
| **ブラウザ** | Chrome, Safari, Firefox, Edge（最新版） |
| **OS** | Windows, macOS, iOS, Android |
| **画面幅** | 320px〜1920px |

### 5.3 SEO対策

- セマンティックHTML
- meta description, OGP設定
- 構造化データ（LocalBusiness, Restaurant）
- sitemap.xml, robots.txt
- Core Web Vitals最適化

### 5.4 アクセシビリティ

- 適切な見出し階層
- alt属性の設定
- フォーカス可視化
- 色のコントラスト比 4.5:1以上

### 5.5 セキュリティ

- HTTPS必須

## 6. デザイン要件

### 6.1 デザインコンセプト

**「モダン × エネルギッシュ」**

- 洗練されたスタイリッシュさ
- 「鉄人」らしい力強さ・活気
- 食欲をそそるシズル感のあるビジュアル

### 6.2 カラーパレット（案）

| 用途 | カラー | 意図 |
|------|--------|------|
| **プライマリ** | 深い赤・オレンジ系 | 食欲、活力、鉄人のイメージ |
| **セカンダリ** | ダークグレー・黒 | 高級感、モダンさ |
| **アクセント** | ゴールド・からし色 | 強調、品質感 |
| **背景** | 白・薄いベージュ | 清潔感、視認性 |

### 6.3 タイポグラフィ

| 用途 | フォント案 |
|------|------------|
| **見出し** | Noto Sans JP Bold / 源ノ角ゴシック |
| **本文** | Noto Sans JP Regular |
| **数字・価格** | Oswald / Roboto |

### 6.4 写真素材

- **素材**: 既存素材を活用
- **加工**: 明るさ・彩度調整でシズル感を強調
- **形式**: WebP推奨（JPEG/PNGフォールバック）

## 9. 運用要件

### 9.1 管理画面（CMS）

**Decap CMS（旧Netlify CMS）を導入**

| 項目 | 内容 |
|------|------|
| URL | https://tetsujin-kujira.com/admin |
| 認証 | GitHubアカウントでログイン |
| 費用 | **無料** |
| 対応デバイス | PC・スマホ（ブラウザのみ） |

#### 9.1.1 スタッフが更新可能な項目

| 項目 | 説明 | 更新頻度 |
|------|------|----------|
| **お知らせ** | 新メニュー、休業日、キャンペーン等 | 月1〜2回 |
| **日替わり弁当（No.1 桜桃）** | 毎月の内容変更 | 月1回 |
| **メニュー価格** | 各弁当の価格変更 | 随時 |
| **メニュー内容** | 各弁当の説明文変更 | 随時 |

#### 9.1.2 更新の流れ

```
1. スマホで /admin にアクセス
2. GitHubアカウントでログイン
3. 編集したい項目を選択
4. フォームで内容を入力・変更
5. 「公開」ボタンで即座に反映
```

※ HTMLやコードの編集は不要。フォーム入力のみ。

### 9.2 バックアップ

- Gitでのバージョン管理（全変更履歴を自動保存）
- 誤って削除しても復元可能

### 9.3 監視・分析

- Google Analytics 4 導入
- Google Search Console 連携

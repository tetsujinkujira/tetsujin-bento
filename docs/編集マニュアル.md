# 鉄人Webサイト 編集マニュアル

このマニュアルでは、スマホやパソコンからWebサイトの内容を更新する方法を説明します。

---

## 編集するファイルは1つだけ！

```
site-config.json
```

このファイルに「お知らせ」と「日替わりメニュー」の両方が入っています。

**編集ページ**: https://github.com/tetsujinkujira/tetsujin-bento/blob/main/public/data/site-config.json

## 1. GitHubにログインする

### ステップ1: GitHubを開く

スマホまたはパソコンのブラウザで、以下のURLを開きます。

```
https://github.com/login
```

### ステップ2: ログインする

- **Username or email**: `tetsujinkujira`
- **Password**: （設定したパスワード）

を入力して「Sign in」ボタンを押します。

---

## 2. ファイルを編集する

### ステップ1: 編集ページを開く

ログインしたら、以下のURLを開きます。

```
https://github.com/tetsujinkujira/tetsujin-bento/blob/main/public/data/site-config.json
```

### ステップ2: 編集モードにする

画面の右上にある **えんぴつマーク** をタップします。

（スマホの場合は、右上の「・・・」メニューの中にあることもあります）

### ステップ3: 内容を書きかえる

ファイルの中身が表示されます。変更したい部分を書きかえてください。

### ステップ4: 保存する

1. 画面を下にスクロールする
2. 「**Commit changes...**」という緑のボタンを押す
3. ポップアップが出たら、もう一度「**Commit changes**」を押す

### ステップ5: 完了！

数分後にWebサイトに反映されます。

## 3. お知らせを変更する

ファイルの **最初の方** にお知らせの部分があります。

```json
{
  "news": {
    "title": "ここがタイトル",
    "date": "2026-01-23",
    "category": "news",
    "body": "ここが本文"
  },
```

### 変更できる場所

| 場所 | 説明 | 例 |
|------|------|-----|
| `title` | お知らせのタイトル | `"本日は臨時休業です"` |
| `date` | 日付（YYYY-MM-DD形式） | `"2026-02-01"` |
| `category` | 種類（下の表を参照） | `"holiday"` |
| `body` | お知らせの本文 | `"誠に申し訳ございません"` |

### カテゴリの種類

| 入力する値 | 意味 |
|-----------|------|
| `"news"` | お知らせ |
| `"new_menu"` | 新メニュー |
| `"holiday"` | 休業日 |
| `"campaign"` | キャンペーン |

### 変更の例：休業日をお知らせしたい場合

```json
  "news": {
    "title": "2月11日は祝日のため休業いたします",
    "date": "2026-02-10",
    "category": "holiday",
    "body": "2月12日より通常営業いたします。ご迷惑をおかけいたします。"
  },
```

## 4. 日替わりメニューを変更する

ファイルの **後ろの方** に日替わりメニューの部分があります。

```json
  "daily": {
    "month": "1月",
    "year": "2026",
    "menuName": "桜桃",
    "menuReading": "サクランボ",
    "price": 700,
    "schedule": [
      ...
    ]
  }
```

### 変更できる場所

| 場所 | 説明 | 例 |
|------|------|-----|
| `month` | 何月のメニューか | `"2月"` |
| `year` | 何年か | `"2026"` |
| `price` | 価格（数字だけ、円マーク不要） | `700` |

### 曜日ごとのおかずを変更する

scheduleの中に、曜日ごとのおかずが書いてあります。

```json
      {
        "day": "月",
        "items": ["ハンバーグ", "コロッケ", "サラダ"]
      },
```

- `day` → 曜日（「月」「火」「水」「木」「金」「土」）
- `items` → その日のおかず（カンマで区切って複数OK）

### 変更の例：2月のメニューに更新

```json
  "daily": {
    "month": "2月",
    "year": "2026",
    "menuName": "桜桃",
    "menuReading": "サクランボ",
    "price": 700,
    "schedule": [
      {
        "day": "月",
        "items": ["チキン南蛮", "きんぴらごぼう", "ほうれん草のおひたし"]
      },
      {
        "day": "火",
        "items": ["さばの味噌煮", "肉じゃが"]
      },
      {
        "day": "水",
        "items": ["生姜焼き", "マカロニサラダ", "煮物"]
      },
      {
        "day": "木",
        "items": ["とんかつ", "ポテトサラダ"]
      },
      {
        "day": "金",
        "items": ["からあげ", "春雨サラダ", "おひたし"]
      },
      {
        "day": "土",
        "items": [],
        "note": "当日、お電話にてご確認ください。"
      }
    ]
  }
```

## 5. よくある間違いと直し方

### 間違い1: カンマを忘れた

**ダメな例：**
```json
{
  "title": "お知らせ"
  "body": "本文"
}
```

**正しい例：**
```json
{
  "title": "お知らせ",
  "body": "本文"
}
```

→ 最後の行以外は、行の終わりに `,` が必要です。

---

### 間違い2: 最後にカンマをつけた

**ダメな例：**
```json
{
  "title": "お知らせ",
  "body": "本文",
}
```

**正しい例：**
```json
{
  "title": "お知らせ",
  "body": "本文"
}
```

→ 最後の行には `,` をつけません。

---

### 間違い3: クォーテーションを忘れた

**ダメな例：**
```json
{
  "title": お知らせ
}
```

**正しい例：**
```json
{
  "title": "お知らせ"
}
```

→ 文字は必ず `"` で囲みます。ただし数字はそのままでOKです。

---

### 間違い4: 全角の記号を使った

**ダメな例：**
```json
｛
  "title"："お知らせ"
｝
```

**正しい例：**
```json
{
  "title": "お知らせ"
}
```

→ `{` `}` `:` `,` `"` `[` `]` は必ず **半角** で入力してください。

## 困ったときは

編集して保存したら、画面が赤くなってエラーが出ることがあります。

その場合は：

1. 落ち着いて、上の「よくある間違い」をチェック
2. わからなければ、変更を元に戻す（ブラウザの「戻る」ボタン）
3. それでもダメなら、担当者に連絡

## ホーム画面にショートカットを追加（おすすめ）

編集ページをスマホのホーム画面に追加しておくと便利です。

### iPhoneの場合

1. Safariで編集ページを開く
2. 下の共有ボタン（□に↑）をタップ
3. 「ホーム画面に追加」をタップ
4. 名前を「鉄人編集」などにして追加

### Androidの場合

1. Chromeで編集ページを開く
2. 右上の「︙」メニュー → 「ホーム画面に追加」

## 2. お知らせを変更する

TOPページに流れる「お知らせ」の内容を変更します。

### ステップ1: ファイルを開く

ログインしたら、以下のURLを開きます。

```
https://github.com/tetsujinkujira/tetsujin-bento/blob/main/public/data/news.json
```

### ステップ2: 編集モードにする

画面の右上にある **えんぴつマーク** をタップします。

（スマホの場合は、右上の「・・・」メニューの中にあることもあります）

### ステップ3: 内容を書きかえる

以下のような画面が表示されます。

```json
{
  "title": "ここがタイトル",
  "date": "2026-01-23T00:00:00.000Z",
  "category": "news",
  "body": "ここが本文"
}
```

#### 変更できる場所

| 場所 | 説明 | 例 |
|------|------|-----|
| `title` | お知らせのタイトル | `"本日は臨時休業です"` |
| `date` | 日付（YYYY-MM-DD形式） | `"2026-02-01T00:00:00.000Z"` |
| `category` | 種類（下の表を参照） | `"holiday"` |
| `body` | お知らせの本文 | `"誠に申し訳ございません"` |

#### カテゴリの種類

| 入力する値 | 意味 |
|-----------|------|
| `"news"` | お知らせ |
| `"new_menu"` | 新メニュー |
| `"holiday"` | 休業日 |
| `"campaign"` | キャンペーン |

#### 変更の例

**休業日をお知らせしたい場合：**

```json
{
  "title": "2月11日は祝日のため休業いたします",
  "date": "2026-02-10T00:00:00.000Z",
  "category": "holiday",
  "body": "2月12日より通常営業いたします。ご迷惑をおかけいたします。"
}
```

### ステップ4: 保存する

1. 画面を下にスクロールする
2. 「**Commit changes...**」という緑のボタンを押す
3. ポップアップが出たら、もう一度「**Commit changes**」を押す

### ステップ5: 完了！

数分後にWebサイトに反映されます。

## 3. 日替わりメニューを変更する

毎月の「桜桃（日替わり弁当）」の内容を変更します。

### ステップ1: ファイルを開く

以下のURLを開きます。

```
https://github.com/tetsujinkujira/tetsujin-bento/blob/main/public/data/daily.json
```

### ステップ2: 編集モードにする

画面の右上にある **えんぴつマーク** をタップします。

### ステップ3: 内容を書きかえる

ファイルの中身はこのようになっています。

```json
{
  "month": "1月",
  "year": "2026",
  "menuName": "桜桃",
  "menuReading": "サクランボ",
  "price": 700,
  "schedule": [
    {
      "day": "月",
      "items": [
        "鶏排（ジーパイ）",
        "麻婆豆腐",
        "わかめ炒め"
      ]
    },
    {
      "day": "火",
      "items": [
        "豚すき煮",
        "コロッケ"
      ]
    }
  ]
}
```

#### 変更できる場所

| 場所 | 説明 | 例 |
|------|------|-----|
| `month` | 何月のメニューか | `"2月"` |
| `year` | 何年か | `"2026"` |
| `price` | 価格（数字だけ、円マーク不要） | `700` |
| `day` | 曜日 | `"月"` `"火"` `"水"` `"木"` `"金"` `"土"` |
| `items` | その日のおかず（複数OK） | 下の例を参照 |

#### おかずの書き方

おかずは `[` と `]` の間に、`"` で囲んで書きます。
複数ある場合は `,` で区切ります。

```json
"items": [
  "ハンバーグ",
  "コロッケ",
  "サラダ"
]
```

    {

        "生姜焼き",
        "マカロニサラダ",

### ステップ4: 保存する

1. 画面を下にスクロールする
2. 「**Commit changes...**」という緑のボタンを押す
3. ポップアップが出たら、もう一度「**Commit changes**」を押す

### ステップ5: 完了！

数分後にWebサイトに反映されます。

## 4. よくある間違いと直し方

### 間違い1: カンマを忘れた

**ダメな例：**
```json
{
  "title": "お知らせ"
  "body": "本文"
}
```

**正しい例：**
```json
{
  "title": "お知らせ",
  "body": "本文"
}
```

→ 最後の行以外は、行の終わりに `,` が必要です。

---

### 間違い2: 最後にカンマをつけた

**ダメな例：**
```json
{
  "title": "お知らせ",
  "body": "本文",
}
```

**正しい例：**
```json
{
  "title": "お知らせ",
  "body": "本文"
}
```

→ 最後の行には `,` をつけません。

---

### 間違い3: クォーテーションを忘れた

**ダメな例：**
```json
{
  "title": お知らせ
}
```

**正しい例：**
```json
{
  "title": "お知らせ"
}
```

→ 文字は必ず `"` で囲みます。ただし数字はそのままでOKです。

---

### 間違い4: 全角の記号を使った

**ダメな例：**
```json
｛
  "title"："お知らせ"
｝
```

**正しい例：**
```json
{
  "title": "お知らせ"
}
```

→ `{` `}` `:` `,` `"` は必ず **半角** で入力してください。

---

## 困ったときは

編集して保存したら、画面が赤くなってエラーが出ることがあります。

その場合は：

1. 落ち着いて、上の「よくある間違い」をチェック
2. わからなければ、変更を元に戻す（ブラウザの「戻る」ボタン）
3. それでもダメなら、担当者に連絡
